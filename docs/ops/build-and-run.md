# Build and Run Guide

## Offline Commute Data Regeneration

The offline commute matrix is generated by [`scripts/generate_travel_data.py`](../../scripts/generate_travel_data.py). The script requires a teammate-local OneMap token in the `ONEMAP_TOKEN` environment variable.

### 1. Get a OneMap Token

1. Create or sign in to your account at the [OneMap API portal](https://www.onemap.gov.sg/apidocs/).
2. Generate a fresh authentication token from the portal.
3. Treat the token as a secret tied to your own account. Do not commit it, paste it into tracked docs, or hard-code it in scripts.

### 2. Set `ONEMAP_TOKEN`

For the current PowerShell session:

```powershell
$env:ONEMAP_TOKEN="paste-your-token-here"
```

To persist it for your Windows user account:

```powershell
setx ONEMAP_TOKEN "paste-your-token-here"
```

For bash or zsh:

```bash
export ONEMAP_TOKEN="paste-your-token-here"
```

After using `setx`, open a new terminal before running the script.

### 3. Install the Script Dependency

```bash
python -m pip install requests
```

### 4. Regenerate the Matrix

Run this command from the repository root:

```bash
python scripts/generate_travel_data.py
```

The script updates [`src/main/resources/commute_data/transit_matrix.csv`](../../src/main/resources/commute_data/transit_matrix.csv) using the repository-tracked rental and destination CSV inputs.

### 5. Troubleshooting

- `ONEMAP_TOKEN environment variable is not set`: export the token in your current shell and rerun.
- `Unauthorized`: generate a fresh token from the OneMap portal and update your environment.
- `Too Many Requests`: wait briefly or increase the delay configured inside the script before retrying.
- Do not store the token in `.env`, source control, screenshots, or committed terminal transcripts.
